Oops! My bad!

Sorry, but this hint is broken. It was generated from irrelevant data. Strictly speaking, it should not have been generated, but it was, so here we are. 

The funny thing is that detecting this situation without going through 
all the rigmarole to build the hints HTML requires us to go through
all the rigmarole to build the hints HTML!

So we do not detect this situation when creating hints because doing so involves thousands of expensive operations, which we do not want to invest our time in at the time; so we just deal with "the odd one" here, where we are doing most of the required work anyway... but if this ever becomes more than "the odd one" then we'll have to wear the cost of detecting these irrelevant bloody hints "properly" upstream.

KRC 2019-12-17 The ExcludedCombosMap now goes most of the way towards detecting and rejecting these irrelevant hints, but it is only a partial solution, which is not a solution, but we'll see this message a lot less often now, which is an improvement, and that's what I am here for.
