HoDoKu ALS-Wing

The three Almost Locked Sets:
ALS A: {0}
ALS B: {1}
ALS C: {2}
share the common values x, y and z such that:
                A
               / \
             x={4} z={6}
             /     \
            C--y={5}--B
Therefore we can remove {3}.

Explanation

The logic is: if z is not in ALS A, then ALS A contains x because only one digit can be removed from an ALS. That means that ALS C has y and thus ALS B has z. The other way round works as well: If ALS B does not contain z then ALS A must. Hence z can be eliminated from any external cells seeing all z's in both ALS A and ALS B.

Note that x and y may not be the same digit.

The restriction in Restricted Common Candidate is that all instances of the value in both ALSs must see each other (be in the same box, row, or col).

An ALS-Wing is an ALS-Chain of length 3. ALS-Wings are handled separately because they are processed differently, primarily in ALS-Chains the RCCs are excluded from the physical overlap, if any, of the two ALSs.
