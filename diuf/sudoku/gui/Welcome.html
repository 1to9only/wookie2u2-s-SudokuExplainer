Intro

Sudoku Explainer (SE) tries to cater for everyone who's interested in Sudoku puzzles, from wee gromets up to Western European university professors, but my focus is on hard (ie beyond a humans ability to solve) puzzles.

Sudoku Explainer shows you how to solve any Sudoku puzzle step by step, explaining each solving technique as it's applied. It also allows you to get all the hints (ie solving techniques) that can be applied to a puzzle, and to apply any or all of them (singularly or multiply).

The Sudoku Solving Techniques herein have been described as "old school". That's fine with me. I've added techniques from HoDoKu, Sukaku/SudokuMonster, and sudopedia.org in order to mitigate this valid criticism. These techniques are NOT faster, they're just (arguably) simpler. My initial focus was TOO performance-centric. The downside is that some of the slower techs are unwanted, by default, so they're there if you're interested, but not slowing you down if you're not.

Sudoku Explainer analyses "the worlds hardest Sudoku puzzle" (according to conceptis.com) in 27 minutes with everything enabled (and unhacked) on my i7, but with my preferred solving techniques selected (see the Solving Techniques section below) it takes about 6 seconds, which is only 270 times faster, so slow Tech selection hurts. Getting Started

If you want help with a puzzle then first you need to punch-it-in to SE. Either left-click on the grey digit (the maybe) to set the cells value, or using the keyboard: navigate to the-cell-to-be-set with the arrow-keys and then press the digit key: 1..9 (Ctrl-1..9 cycles a maybe).

Spacebar, delete, backspace, or 0 (zero) all clear the current cell value. If you stuff-up then Ctrl-Z (undo) goes back one step at a time. Ctrl-Y to redo.

When you've got your whole puzzle punched-in press Ctrl-S to save. Now's a good time to double-check that the puzzle is correct, before you start solving a typo. A reload could be handy later, so don't forget to save. Ctrl-S now.

Press F7 to validate: ie check that this puzzle has one and only one solution. I've seen insufficient clues in published Sudoku's; just some moron rushing to meet a publication deadline, armed with zero understanding of Sudoku, the force, algebra, and/or the ability to nut-out how to tie ones own shoelaces.

If you hold down the left-mouse-button over the row labels it highlights all occurrences of that number, making patterns a bit easier to see. Drag the row label into the grid to retain highlights whilst doing other mousy stuff.

If you get stuck press F5 for a hint. To apply that hint and get the next one just press enter. The enter key only works reliably (one tries and sometimes fails) when the hintsTree (top right) is selected. Did I mention that GUIs just s__t me?

Anyway, that's probably enough to get you started. Have fun with it. Come back and read the advanced stuff when you get bored.

Advanced Stuff

More "advanced" Sudokuists will want to remove maybes, aka potential values, or candidates. That's those small grey digits in the empty cells. If you don't see any small grey digits goto the Options menu and select "Show maybes".

To remove a maybe right click on the small grey digit; or using the keyboard: select the cell with the arrow keys and then Ctrl-digit. Do the same to restore a maybe; right-click where it should be. You can't restore an invalid one.

You'll notice that Sudoku Explainer enforces the basic rule of Sudoku: "There will be one-and-only-one instance of each value in each region". So when you punch a 7 into a cell then all the 7's disappear from the maybes of other cells in the box (the 3x3 block), row (horizontal), and col (vertical), and no matter what you do you can't get them back, unless you zap that 7.

To zap the 7 click on the cell and press the 0 key, or delete, or the space bar.

Sudoku puzzle books are all a bit soft (it's an expert scale) so just seeing the maybes could get you unstuck; but if not then press F5 to find a hint. Then you can just press enter to apply that hint and get next, and next... Solvered!

Menu stuff worth a mention

  • File menu ~ load next Puzzle or just Ctrl-P loads the next puzzle. The meaning of "next" depends on what you've got open. The .mt files (magic-tour) are a multi-puzzle format, but next on a .txt file opens the next text-file in the current directory. NOTE that when you open the next puzzle SE analyses the puzzle automatically, which may take a while, depending on the difficulty of the puzzle and which Solving Techniques you've got enabled (some are MUCH slower than others).
  • File menu ~ Generate to generate new Sudoku puzzles. I warn you that my generate is poor compared to the best, but it's good for simpler puzzles.
  • File menu ~ Recent Files to go back in time.
  • File menu ~ Reload (Alt-F-R) or just Ctrl-R reloads the saved puzzle from disk, which I find handy.
  • File menu ~ log View for techies: view hints of a type in the .log of a previous LogicalSolverTester run, to re/exercise the hinter/s and/or see how hinttype/s look in GUI.
  • Tools menu ~ Reset Potential Values restores the puzzles maybes, which is handy if you bugger-it-all-up and want to restore the puzzle to a valid state before trying again. Note that reset doesn't remove cell values, even if they're wrong. Use Ctrl-Z or Reload for that.
  • F6 - Get a clue points you towards the next step. Shift-F6 gives you a bit more information (it's basically the hint for simpler techniques). For harder techniques a clue is the help I think I should want when solving a Sudoku manually because it tells me which technique to apply next, allowing me to skip all the inapplicable techniques; but what I actually use is the hints, but I'm a programmer, so for me the challenge is writing the code to produce the hints to solve all Sudoku puzzles, not solving A puzzle, as I presume it is for most people. Clues are there if you want them.
  • F7 - Check validity checks that there are no doubled values; that each empty cell has at least 1 potential value; that all regions have a possible location for each unplaced value; and finally that the Sudoku has a unique (one and only one) solution. This is pretty handy. I'm not really surprised (just disgusted) at the prevalence of blatant basic invalidities (like 16 clues) in published Sudokus, because idiots are always in a hurry. Quality takes time. Get over it.
  • F9 - Analyse tells you how hard the puzzle is, and summarises the steps used to solve it. Note that this is far from the only possible solution to the puzzle, it's just a (not THE) simplest recipe using the enabled Solving Techniques. There may be other shorter recipes that don't increase the difficulty. I'm not seeking the shortest simplest path, just the simplest; because that's simpler. Yeah? A*ing chumpy edges scares me.

Selecting Solving Techniques

  • Options ~ Solving techniques... allows you to select which Sudoku solving techniques are considered when seeking the next hint, and are also used in F9 Analyse:
    • Naked and Hidden Singles are mandatory, so there CheckBoxes are checked and disabled. It makes eff-all sense trying to solve Sudokus without them, so they're mandatory.
    • Dynamic Plus is a bit of a catch-all. It only misses on expert puzzles. Nested Unary is slower, but it ALWAYS hints, so I use both as catch-alls, that is I always have them both enabled. The remainder of Nested* can go. They're never executed unless you Shft-F5, to get MORE hints.
    • Direct* are useless, but which I mean when unwanted ALL hints are still found by there non-direct equivalents. Direct just runs the same algorithm twice on each pass, to show hintier ones sooner, which is really just a waste of time, unless it isn't because that's what you really want.
    • Choosing BUG, Coloring, XColoring, Medusa3D, and/or GEM (Graded Equinox Marks) is pretty complicated, so here's the break down:
      • BUG (Bivalue Umbilical Grave) is a Grand-daddy solving technique; which might reasonably now be regarded as "deprecated" because it ambulates with a Zimmer frame, but it still sort-of-works, sometimes, eventually.
      • Coloring does Simple (2 sets) and Multi (3+ sets) Coloring . It's faster and finds more hints than BUG. It's the ONLY coloring technique searching multiple (3+) color-sets. If Coloring is unwanted then multiple color-set hints are be found by the Unary Chainer, if wanted. Multi-Coloring hints are NOT sought by any of XColoring, Medusa3D, or GEM. Coloring implements the ONLY Multi-Coloring. You should use Coloring!
      • XColoring (Extended Coloring) finds more Simple Coloring hints, some of which GEM misses, so I still use it. XColoring, being more complex, is a bit slower than Coloring. So be it.
      • Medusa3D finds even more Simple Coloring hints than XColoring, using value-cell sets, but is thereby a bit slower again. Medusa3D finds a subset of GEM hints, and finds less eliminations in the same grid so Medusa3D is counter-productive with GEM.
      • GEM (Graded Equivalence Marks) is the "ultimate" in Simple Coloring. It finds a superset of Medusa3D's hints, and is actually a bit faster than Medusa3D in this implementation, but is MUCH faster on a ns/elim basis coz it finds so-many-more cells to set.
      • So to find the simplest hint as quickly as possible I recommend using Coloring, XColoring, and GEM together:
        • Coloring does the ONLY Multi-Coloring.
        • XColoring hits a few that GEM (by design) misses.
        • GEM for ultimate Simple-Coloring.
        • but any combo of BUGs/Colorings are fast enough in practice.
      • Opportunities for improvement:
        • The GEM spec says the color-chart is re-usable; which I have not implemented, coz I'm lazy, so there's an opportunity.
        • A public color-chart could also be used as a trip-wire-network: if any fully colored cell-value is set subsequently then all cell-values and ons of that color could also be set, but I haven't, to keep it simple.
    • Naked Pent and Hidden Pent are both degenerate, which means that every hint is the agglomerate of two simpler techniques. In practice this means they don't hint, because we execute solving techniques in (approximate) increasing order of complexity.
    • Finned Jellyfish are just a tad slow, so I usually drop them, but you can use them if you're more patient than I am. Most people are. sigh.
    • Mutant Swampfish is degenerate to Finned and Franken Swampfish, so if you really want Mutant Swampfish then unwant Finned and Franken Swampfish, but be warned that Mutant means slow, and it finds no more Swampfish; so I never use Mutant anything, nor Krakens, for basically the same reason.
      • The same partially applies to Mutant Swordfish and Mutant Jellyfish. Mutant searches all Finned and Franken combos with the Mutant-only combos.
      • if you want Mutants then drop Finned and Franken, but be prepared to wait.
      • I use Finned and sometimes Franken. Bugger the Mutant and his Kraken!
