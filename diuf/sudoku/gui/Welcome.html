Intro

Sudoku Explainer tries to cater for everyone who's interested in Sudoku puzzles, from wee gromets up to Western European university professors, but the focus is on hard (ie beyond a humans ability to solve) puzzles.

The Techniques herein have been described as "old school". That's fine with me. I've added techniques from HoDoKu and Sukaku in order to mitigate this valid criticism. They're not faster, they're just simpler. My initial focus was TOO performance-centric. The downside is you'll probably want to unwant slow techs.

Sudoku Explainer shows you how to solve any Sudoku puzzle step by step, explaining each solving technique as it's applied. It also allows you to get all the hints (ie solving techniques) that can be applied to a puzzle, and to apply any or all of them (singularly or multiply).

Sudoku Explainer solves "the worlds hardest Sudoku puzzle" according to conceptis.com in 125.610 seconds with all A*E's enabled and unhacked on my i7 (see the Solving techniques section below for my config preferences): 

8..........36......7..9.2...5...7.......457.....1...3...1....68..85...1..9....4..
(copy-and-paste this puzzle (Edit menu ~ Paste grid) then press F9 to analyse).

Getting Started

If you just want a bit of help with a puzzle then the first step is punching it in. Either left-click on the small grey digit (the maybe) so set the cells value, or with the keyboard: navigate to the cell-to-be-set with the arrow-keys and then press the digit key: 1..9.

Spacebar, delete, backspace, or 0 (zero) all clear the current cell value. If you stuff-up then Ctrl-Z goes back one step at a time. Ctrl-Y does a redo.

When you've got your whole puzzle punched-in do NOT forget to Ctrl-S for save BEFORE you start solving it. A reload might be handy. I always forget to save. Sigh.

If you get stuck press F5 for a hint. To apply that hint and get the next one just press enter.

Anyway, that's probably enough to get you started. Have fun with it. Come back and read the advanced stuff when you get bored.

Advanced Stuff

More "advanced" Sudokuists will want to remove maybes (potential values). That's those small grey digits in the empty cells. To do that just right-click on the small grey digit, or with the keyboard: select the cell with the arrow keys and then Ctrl-digit to remove/restore the maybe. You can't restore an illegal maybe.

You'll notice that Sudoku Explainer enforces the basic rule of Sudoku: "There will be one-and-only-one instance of each value in each region". So when you punch a 7 into a cell then all the 7's disappear from the other cells in it's box (the 3x3 thingummy), row (horizontal), and col (vertical).

Sudoku puzzle books are all a bit soft (it's an expert scale) so just seeing the maybes could get you unstuck; but if not then press F5 to find a hint. Then you can just press enter to apply that hint and get the next one, and the next, and the next... Sudoku Solvered! Too Bloody Easy!

The following "frequently used" buttons are arranged along the bottom of the form (I never use the buttons. I do hope they still work):

  • Clear hints clears the hints tree and refreshes the Sudoku grid.
  • Solve step applies the selected hint (if any) and gets the next (simplest possible) step in solving the puzzle. Click the [Solve step] button again to apply the selected hint, and get the next one; or just double-click on the hint in the hints tree, or click in the hintsTree (to select it) and press enter. That's how to watch the Sudoku puzzle being solved: Just Keep Pressing Enter! which takes all the fun out of it really. Sigh. Except that SE solves any Sudoku puzzle, even really really really hard ones (like conceptis's above), and my challenge is to solve all possible Sudoku puzzles using logic, and to do so faster than everybody-else, mainly because whacking-off gets boring eventually, I had superfluous hair (sigh), and it was there.
  • Get next hint to get hints again without applying the current one. Why would you do that? Because there's a button for it, obviously!
  • Apply hint to apply the current hint, without getting the next one. You can select multiple hints in the HintsTree and apply them all with this button; or Just Press Enter! Dunno why you'd actually do that, but it is possible, because there's a button for it, which is nice really, isn't it.
  • Get all hints which can be applied to the current grid. Hold down the shift key to get even more hints, but be forewarned, this can take a a minute or three on my i7. The HintsTree is grouped by category & hinter, in increasing order of difficulty (now increasing eliminations). You'll need to expand each category by clicking on it. Be warned that 20,000+ hints may be produced, so the GUI bogs, especially when "Filter hints" is NOT selected. It filters with the old "does it remove any NEW maybes" trick. So if all of a-more-complex-hints actions (cells set & maybes eliminated) are already covered by a simpler hint then the latter is suppressed, which keeps the hint-count down, so the GUI doesn't bog.

Some other stuff also worth a mention:

  • File menu ~ load next Puzzle or just Ctrl-P loads the "next" puzzle. The meaning of "next" depends on what you've got open. The .mt files (magic-tour) are a multi-puzzle format, but next on a .txt file opens the next text-file in the current directory. NOTE that when you open the next puzzle Sudoku Explainer analyses the puzzle automatically, which may take a while, depending on the difficulty of the puzzle and which Solving Techniques you've got enabled.
  • File menu ~ Reload (Alt-F-R) or just Ctrl-R reloads the saved puzzle from disk. I find this really handy. It'll even let you try chaining manually, if you're over 90, have a decade to kill, and are SERIOUSLY sexually frustrated but find that you no longer have the manual dexte.... Never mind. Ctrl-R to reload the puzzle. OK?
  • File menu ~ log View for techies: view hints of a type in the .log of a previous LogicalSolverTester run, to re/exercise the hinter/s and/or see how hinttype/s look in GUI.
  • Tools menu ~ Reset Potential Values restores the maybes in the current grid, which is handy if you bugger-it-all-up and want to restore the grid to a valid state before trying again. But note that reset PV's doesn't remove cell values, even if they're wrong.
  • F6 - Get a clue points you towards the next step. Shift-F6 gives you a bit more information (it's basically the hint for simpler techniques). For harder techniques a clue is the help I think I should want when solving a Sudoku manually because it tells me which technique to apply next, allowing me to skip all the inapplicable techniques; but what I actually use is the hints, but I'm a programmer, so for me the challenge is writing the code to produce the hints to solve all Sudoku puzzles, not solving A puzzle, as I presume it is for most people. Clues are there if you want them.
  • F7 - Check validity checks that there are no doubled values; that each empty cell has at least 1 potential value; that all regions have a possible location for each unplaced value; and finally that the Sudoku has a unique (one and only one) solution. This is pretty handy. I'm not really surprised (just disgusted) at the prevalence of blatant basic invalidities (like 16 clues) in published Sudokus, because morons are always in a hurry.
  • F9 - Analyse tells you how hard the puzzle is, and gives you a summary of steps which may be used to solve it. Note that this is far from the only solution to the puzzle, it's just a (not THE) simplest recipe with the enabled Solving Techniques. There may be other shorter recipes that don't increase the difficulty. I'm not seeking the shortest simplest path, just the simplest; because that's simpler. Yeah? A real guru could do better, I'm sure. Contemplating A*ing edges this meaty scares me. sigh.
  • If you want an itemised solution then run the LogicalSolverTester over the puzzle and look at the log file. That's why I haven't bothered with it in the GUI. I figure most users don't want it, and those that do also want a batch.
  • Options ~ Solving techniques... allows you to select which Sudoko solving techniques are applicable:
    • Naked Singles, Hidden Singles, Dynamic Plus and Nested Plus are mandatory, so there CheckBoxes are checked and disabled.
    • You can't disable singles because they're "core" techniques. It makes no sense to try to solve a Sudoko without them.
    • Dynamic Plus is a catch-all. It's a bit slow, but misses only on the hardest puzzles. Nested Plus is really slow, mandatory, and ALWAYS hints; in fact Nested Unary (faster) also never fails to find a hint, so maybe it should be the catch-all instead, except that'd preclude Nested Multiple and Nested Plus, where the real disenhirsuterative fun is. sigh.
    • Direct* are useless, but which I mean when they're unchecked the hints are still found by there non-direct equivalents. Direct just runs the same algorithm twice on each pass, which is a waste of time.
    • Naked Pent and Hidden Pent are both degenerate, which means that every hint is the agglomerate of two simpler techniques. In practice this means they don't hint, because we execute solving techniques in (approximate) increasing order of complexity.
    • Mutant Swampfish is degenerate to Finned and Franken Swampfish, so use either (Finned Swampfish and Franken Swampfish) OR Mutant Swampfish, but not all three, or Mutant Swampfish finds no hints, which is a waste of time.
    • The same sort-of-applies to Mutant Swordfish and Mutant Jellyfish. All Finned and Franken hints are found by the Mutant hinters, along with the Mutant-only combinations, because filtering them out is slower; so if you want the Mutants you can drop Finned and Mutant. Just be prepared to wait.
    • BE WARNED: Mutant Jellyfish are TOO SLOW (about 20 minutes for top1465).
    • Aligned Ally (one up from Diagon Ally;-):
      • Aligned Pair is fast,
      • Aligned Triple is fast enough,
      • Aligned Quad is acceptable,
      • Aligned Pent is acceptable (I go unhacked),
      • Aligned Hex is acceptable (I go unhacked),
      • Aligned Sept is too slow unhacked, and acceptable hacked but then hints very rarely. sigh.
      • BE WARNED: Unhacked Aligned Oct, Nona and Dec are so slow there's an APB out on them from the School of Algorithmic Incontinence at the Walgatt College of Ear Wax Removal and Advanced Masturbation. My advise is don't use A8+E. They're as rare as chickens lips anyways.
      • BE WARNED: The "correct" Aligned Dec runs for a ____ING DAY!!!!
      • Suggestions on AE'ing faster are welcome. City life's made him a bit slow, Redge.
    • Whips & Chainers:
      • The Chainers are now so fast compared to the AE debacle that I really don't know why you'd disable them, except maybe Nested* if you're impatient with Shift-F5. I use all Chainers, but not Nested*.
      • Unary Chain would be better named Unary Chains and Bidirectional Cycles, but that's just too bloody long. Dynamic Chain finds Unary Chains, but not Bidirectional Cycles, which are "pretty rare" anyways.
      • Nishio Chain is when an assumption has contradictory effects, hence the assumption must be wrong.
      • Multiple Chain starts from a cell with two or more potential values. If you unwant Multiple then Dynamic takes longer, which is faster overall; but there's no distinction between static and dynamic chaining, so hints are incorrectly rated harder than they really are.
      • Dynamic Chain removes maybes from the grid to track effects of each assumption, playing "who's your daddy" on Ass. sigh. Dynamic Chain finds Unary Chains (but not Cycles) and Multiple Chains. I use Dynamic.
      • Dynamic Plus applies Four Quick Foxes (Locking, NakedPair, HiddenPair, Swampfish) on top of dynamic chains.
        This only fails to find a hint on expert level puzzles. Normal people can rely on it to always find a hint. It's check box is disabled so you can't disable it, unless you're a programmer.
      • Nested Unary always (AFAIK) hints, even on the worlds hardest puzzle (above). "Normal" people can disable it with confidence.
      • Nested Multiple/Dynamic/Plus Chains aren't ever used in anger. They exist solely for the amusement of afficionados. You can see them with Shift-F5; just be patient.
      • I normally run with Nested Unary/Multiple/Dynamic disabled, but Nested Plus is always enabled, as the ultimate catch-all.

I should also mention the LogicalSolverTester, which solves a MagicTour (.mt) file containing multiple (upto 9999) puzzles, just to see how quickly we can do it; and verbose logging shows all the hints used to solve each puzzle, which is how I find puzzles for my test-cases.

My "standard" test data file is top1465.d5.mt. It contains 1465 Sudoku puzzles in "approximate" order of hardest to easiest. I need a variety of puzzles to exercise my hinters, and a "standard" for comparative performance testing.
