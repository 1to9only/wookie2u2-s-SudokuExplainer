Intro

Sudoku Explainer (SE) tries to cater for everyone who's interested in Sudoku puzzles, from wee gromets up to Western European university professors, but my focus is on hard (ie beyond a humans ability to solve) puzzles.

Sudoku Explainer shows you how to solve any Sudoku puzzle step by step, explaining each solving technique as it's applied. It also allows you to get all the hints (ie solving techniques) that can be applied to a puzzle, and to apply any or all of them (singularly or multiply).

The Sudoku Solving Techniques herein have been described as "old school". That's fine with me. I've added techniques from HoDoKu, Sukaku/SudokuMonster, and sudopedia.org in order to mitigate this valid criticism. These techniques are NOT faster, they're just (arguably) simpler. My initial focus was TOO performance-centric. The downside is that some of the slower techs are unwanted, by default, so they're there if you're interested, but not slowing you down if you're not.

Sudoku Explainer analyses The Worlds Hardest Sudoku Puzzle according to conceptis.com in 27 minutes with everything enabled (and unhacked) on my i7, but with my preferred solving techniques "wanted" it takes just 4.3 seconds, which is only 373 times faster, so slow Techs hurt.

Getting Started

If you want help with a puzzle then first you need to punch-it-in to SE. Either left-click on the grey digit (the maybe) to set the cells value, or using the keyboard: navigate to the-cell-to-be-set with the arrow-keys and then press the digit key: 1..9 (Ctrl-1..9 cycles a maybe).

Spacebar, delete, backspace, or 0 (zero) all clear the current cell value. If you stuff-up then Ctrl-Z (undo) goes back one step at a time. Ctrl-Y to redo.

When you've got your whole puzzle punched-in press Ctrl-S to save. Now's a good time to double-check that the puzzle is correct, before you start solving a typo. A reload could be handy later, so don't forget to save. Ctrl-S now.

Press F7 to validate: ie check that this puzzle has one and only one solution. I've seen insufficient clues in published Sudoku's; just some moron rushing to meet a publication deadline, armed with zero comprehension of Sudoku, algebra, logic, the force, and/or the ability to workout how to tie ones own shoelaces.

If you hold down the left-mouse-button over the row labels it highlights all occurrences of that number, making patterns a bit easier to see. Drag the row label into the grid to retain highlights whilst doing other mousy stuff.

If you get stuck press F5 for a hint. To apply that hint and get the next one just press enter. The enter key only works reliably (one tries and sometimes fails) when the hintsTree (top right) is selected. Did I mention that GUIs just s__t me?

Anyway, that's probably enough to get you started. Have fun with it. Come back and read the advanced stuff when you get bored.

Advanced Stuff

More "advanced" Sudokuists will want to remove maybes, as in cell "may be" this value. Maybes are also commonly called potential values, or candidates. No matter what you call them, they're displayed as small grey digits in the "empty" cells. An empty cell is a cell whose value is yet to be determined. If you don't see any small grey digits goto the Options menu and select "Show maybes".

To remove a maybe right click on the small grey digit; or using the keyboard: select the cell with the arrow keys and then Ctrl-digit. Do the same to restore a maybe; right-click where it should be. You can't restore an invalid one.

You'll notice that Sudoku Explainer enforces the basic rule of Sudoku: "There will be one-and-only-one instance of each value in each region". So when you punch a 7 into a cell then all the 7's disappear from the maybes of other cells in the box (the 3x3 block), row (horizontal), and col (vertical), and no matter what you do you can't get them back, until you zap that 7, when they all reappear magically.

To zap the 7 click on the cell and press the 0 key, or delete, or the space bar.

Sudoku puzzle books are all a bit soft (it's an expert scale) so just seeing the maybes could get you unstuck; but if not then press F5 to find a hint. Then you can just press enter to apply that hint and get next, and next... Solvered!

FYI, my mom does Sudokus. Occasionally she get's stuck. I punch it in and step through the solution, and then mark-up her Sudoku book with any hints which are harder than HiddenPair. That's usually enough to get her going again. She has no idea how much harder Sudokus can get than the published garbage (nothing above medium, ergo you're still on your P's in the expert world).

Selecting Solving Techniques

  • Options ~ Solving techniques... allows you to select which Sudoku solving techniques are used to find hints, and in F9 Analyse, and the batch (see below):
    • Naked and Hidden Singles are mandatory, so there CheckBoxes are checked and disabled. It just makes eff-all sense trying to solve Sudokus without singles, so they're both mandatory.
    • Direct* are useless, but which I mean when unwanted ALL hints are still found by there non-direct equivalents. Direct just runs the same algorithm twice on each pass, to show hintier ones sooner, which is really just a waste of time, unless it isn't because that's what you want.
      Note: Direct* hinters are NEVER used in the batch, even when wanted in GUI.
    • Choosing Coloring, XColoring, Medusa3D, and/or GEM (Graded Equivalence Marks) is a bit complicated, so here's a break down:
      • BUG (Bivalue Universal Grave) is history. It has been removed. He was too slow.
      • Coloring does basic SimpleColoring and the ONLY MultiColoring. SimpleColoring uses 2 coloring sets; and MultiColoring uses 3-or-more coloring sets. If Coloring is unwanted then multiple color-sets are found by Unary Chainer. MultiColoring (3+) are NOT sought by any of XColoring, Medusa3D, or GEM. Coloring does the ONLY MultiColoring, so I always use it.
      • XColoring (Extended Coloring) finds more Simple Coloring hints, some of which GEM misses, by design, so I always use it. XColoring, being a bit more complex, is a bit slower than Coloring. Meh!
      • DROP Medusa3D. It finds a subset of GEM hints and will produce less elims in the same grid, which then prevents GEM finding the hint, hence Medusa3D is actually counter-productive when used with GEM. Medusa3D is SimpleColoring with extra rules on top of XColoring, to find more elims, but still not quite as many as GEM. Don't use it!
      • GEM (Graded Equivalence Marks) is the "ultimate" in SimpleColoring. It finds a superset of Medusa3D's hints. It's a bit faster than Medusa3D in this implementation, but is MUCH MUCH faster on a ns/elim basis because it finds MANY MANY more cells to set.
      • So to find the simplest hint as quickly as possible I use Coloring, XColoring, and GEM together:
        • Coloring does the ONLY MultiColoring.
        • XColoring finds a few that GEM (by design) misses.
        • DROP Medusa3D coz it impinges some GEM hints.
        • GEM for the ultimate SimpleColoring.
        • Note that GEM should find all SimpleColoring, but Coloring finds the basic ones faster, and counter-intuitively GEM's performance actually improves when you present it with a grid that contains no basic coloring problems. An observation for which I'm too dumb to see an explanation.
        • Any combinations of Coloring/s is fast enough in practice.
